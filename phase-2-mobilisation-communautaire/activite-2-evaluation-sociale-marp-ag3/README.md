# Activité 4 — Troisième AG : évaluation sociale participative / Participatory social assessment (PRA/MARP)

> Phase 2 · Activité 2 · Étape 4 · Source : Part 1, §4

## Quick reference (EN)

A **participatory social assessment** empowers the community through awareness, informed decision-making and leadership. The community gathers and analyses information on its needs/problems and assets/resources to understand its situation, then decides what to do — using its own resources plus outside support. This builds the **Community Development Plan (PDC)**. Full participation is essential: it produces good information and a strong sense of ownership. **The community itself does the analysis** — the facilitator records decisions but does not decide for them.

It uses **MARP/PRA tools** as starting points for discussion (not ends in themselves). Five tools: **Social/resources/hazards map · Gender asset analysis · Seasonal calendar · Venn diagram · Historical natural-resource profile.** Tools are run in **focus groups** (women, herders/farmers, opinion leaders/chiefs, youth, vulnerable people), then reported and discussed in plenary.

**Duration:** 1–3 days starting at the 3rd village meeting. **Scenario check first:** if a valid participatory assessment/PDC exists from the last 3 years (≥50% participation, women involved, other vulnerable groups involved, full transparent discussion, climate included), the village may reuse/confirm it (Scenario 1); otherwise run the full process (Scenario 2).

Sequence inside the assessment: present objectives + future climate scenarios → form 5 focus groups → run MARP tools + discuss each → report to plenary → list problems/challenges → vision (5 years) → list development activities → prioritize in groups. The outputs of the tools, focus-group discussions and ranking together form the PDC.

## Tâches du facilitateur (checklist)

- [ ] Préparer les données climatiques accessibles.
- [ ] Vérifier le scénario 1 / scénario 2.
- [ ] Former les 5 focus groupes ; faciliter les 5 outils MARP + discussions.
- [ ] Faire rapporter chaque groupe en plénière ; résumer.
- [ ] Conduire : liste des problèmes → vision → activités → priorisation (groupes puis plénière).
- [ ] Indiquer l'origine des priorités (femmes/jeunes/tous) pour l'application DCC.
